[=] Assignment

Definition

Assigns a computed value to a variable in the current environment.

Syntax

[=]-[name][expression]

Immutable Assignment

*[=]-[name][expression]

The * prefix defines an immutable variable that cannot be reassigned.

Semantics

  • The expression is evaluated before assignment
  • The resulting value is stored under name
  • If the variable exists:
    • Mutable → overwritten
    • Immutable → runtime error
  • Variable names are treated as identifiers (no quotes)

Type Behavior

Variables are dynamically typed. The assigned value determines the variable's type.

Examples

[=]-[a][10]
[=]-[b][${[+]-[a][5]}]

*[=]-[c][100]    // immutable

Notes

  • No implicit type conversion occurs during assignment
  • Variables must be defined before use
  • Assignment does not produce output
