[/CNV/] Conversion

Definition

Performs explicit type conversion from one type to another.

Syntax

[/CNV/]:[FROM]:[TO]-[expression]

Semantics

  • The expression is evaluated first
  • The result must match the declared FROM type
  • Conversion is applied to produce the TO type
  • If mismatch or invalid conversion → runtime error

Supported Conversions

INT ↔ DEC
INT ↔ STR
STR ↔ DEC
STR → BOO
BOO → STR

Detailed Behavior

STR → INT

  • String must contain only digits

STR → DEC

  • Must be valid numeric format

STR → BOO

  • Only "TRUE" or "FALSE" allowed

INT/DEC → STR

  • Converted using standard string representation

INT ↔ DEC

  • INT → DEC preserves value
  • DEC → INT truncates fractional part

Examples

[/CNV/]:[STR]:[INT]-["123"] → 123
[/CNV/]:[INT]:[STR]-[10] → "10"
[/CNV/]:[STR]:[BOO]-["TRUE"] → TRUE
[/CNV/]:[DEC]:[INT]-[10.7] → 10

Notes

  • All conversions are explicit
  • No implicit coercion exists in the language
  • Conversion must match declared source type exactly
