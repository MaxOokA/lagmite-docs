Lagmite Language Specification

1. Introduction

Lagmite is a symbolic, expression-oriented programming language. All computations are represented as bracketed operator expressions. Execution proceeds sequentially, evaluating one statement per line.

2. Syntax

2.1 General Form

[OPERATOR]-[ARG1][ARG2]

2.2 Nested Expressions

${ expression }

2.3 Literals

  • Integer: 10
  • Decimal: 10.5
  • String: "text"
  • Boolean: TRUE, FALSE

3. Types

Lagmite defines four primitive types:

  • INT — Integer
  • DEC — Decimal
  • STR — String
  • BOO — Boolean

3.1 Type Promotion

Operations involving INT and DEC yield DEC unless both operands are INT.

4. Evaluation

Expressions are evaluated recursively. Sub-expressions inside ${...} are evaluated before the enclosing expression.

Evaluation is strict and does not perform implicit conversions.

5. Operators

5.1 Arithmetic

+ : (INT|DEC, INT|DEC) → INT|DEC
- : (INT|DEC, INT|DEC) → INT|DEC
* : (INT|DEC, INT|DEC) → INT|DEC
/ : (INT|DEC, INT|DEC) → DEC
% : (INT, INT) → INT
^ : (INT|DEC, INT|DEC) → INT|DEC
R : (INT|DEC, INT|DEC) → INT|DEC

Special cases:

  • + on STR performs concatenation
  • * supports STR × INT repetition
  • + on BOO performs logical OR

5.2 Boolean

&    : (BOO, BOO) → BOO
!&   : (BOO, BOO) → BOO
!+   : (BOO, BOO) → BOO
X+   : (BOO, BOO) → BOO
X!+  : (BOO, BOO) → BOO
!    : (BOO) → BOO

5.3 Comparison

>  : (NUM, NUM) → BOO
<  : (NUM, NUM) → BOO
>= : (NUM, NUM) → BOO
<= : (NUM, NUM) → BOO
== : (T, T) → BOO
!= : (T, T) → BOO

Where NUM ∈ {INT, DEC}

6. Conversion

6.1 Syntax

[/CNV/]:[FROM]:[TO]-[value]

6.2 Valid Conversions

INT ↔ DEC
INT ↔ STR
STR → BOO
STR ↔ DEC

Conversions are explicit and must match declared source type.

7. Variables

7.1 Assignment

[=]-[name][value]

7.2 Immutable Assignment

*[=]-[name][value]

Immutable variables cannot be reassigned after initialization.

7.3 Lookup

Variables are referenced by name.

8. Errors

Lagmite implementations must produce a general error message for all runtime and validation failures. Specific error diagnostics are not exposed.

Execution terminates immediately upon encountering an error.
