Lagmite Language Documentation

Overview

Lagmite is a symbolic, bracket-based programming language focused on strict structure and clarity. All operations are written in a consistent format and evaluated line-by-line.

Core Syntax

All operations follow this structure:

[OPERATOR]-[argument1][argument2]
[=]-[a][10]       // assign variable
*[=]-[b][20]      // immutable variable
[/DSP/]-[a]       // display value

Data Types

  • INT → Integer
  • DEC → Decimal
  • STR → String
  • BOO → Boolean (TRUE / FALSE)

Arithmetic Operations

[+]-[a][b]
[-]-[a][b]
[*]-[a][b]
[/]-[a][b]
[%]-[a][b]
[^]-[a][b]
[R]-[n][x]   // nth root

Boolean Operations

[&]-[a][b]
[!&]-[a][b]
[!+]-[a][b]
[X+]-[a][b]
[X!+]-[a][b]
[!]-[a]

Comparison Operations

[>]-[a][b]
[<]-[a][b]
[>=]-[a][b]
[<=]-[a][b]
[==]-[a][b]
[!=]-[a][b]

Type Conversion

Format:

[/CNV/]:[FROM]:[TO]-[value]
[/CNV/]:[INT]:[DEC]-[a]
[/CNV/]:[STR]:[INT]-["123"]

Nested Expressions

Use ${...} to evaluate expressions inside expressions:

[/DSP/]-[${[+]-[a][b]}]

Example Program

[=]-[a][10]
[=]-[b][20]
*[=]-[c][5]

[/DSP/]-[${[+]-[a][b]}]
[/DSP/]-[${[*]-[a][c]}]

Rules & Notes

  • All syntax must follow strict bracket structure
  • Variables must be defined before use
  • Immutable variables cannot be reassigned
  • Expressions must have correct argument counts
  • Conversions must be valid

Error Handling

Lagmite uses a unified error message system. Instead of detailed diagnostics, a general message is displayed whenever an error occurs.
